## КАРТА ПРОБЛЕМ — ЧТО СЛОМАНО

```
🔴 КРИТИЧЕСКИЕ (могут привести к потере данных/денег)
├── 1. Нет backend — вся логика в браузере, обход через DevTools
├── 2. TOCTOU уязвимость в transactionService.update/delete (379 строк)
├── 3. Закупки НЕ АТОМАРНЫ — purchaseService.add не обновляет склад
├── 4. Удаление заказа НЕ реверсит склад и долги
├── 5. handleSave* баг — в 5 из 8 обработчиков updates теряются при add
├── 6. Нет backup — потеря данных = потеря бизнеса
└── 7. retained earnings = затычка (баланс ВСЕГДА сходится)

🟠 ВЫСОКИЕ (ограничивают рост и безопасность)
├── 8. Потолок ~50 пользователей (все данные в памяти)
├── 9. onSnapshot без пагинации → bandwidth explosion
├── 10. Нет мониторинга/алертинга — слепое управление
├── 11. Нет двойной записи (General Ledger) — не ERP
├── 12. Нет периодов учёта (accounting periods)
├── 13. Любой авторизованный читает ВСЕ данные (зарплаты, ИНН)
├── 14. Нет блокировки уволенных (token revocation)
└── 15. App.tsx God Component (641 строк, 14 хуков)

🟡 СРЕДНИЕ (технический долг)
├── 16. Sales компонент: 18 props → prop drilling
├── 17. JSON.stringify для diff detection → неэффективно
├── 18. Нет .env.example → проблема onboarding
├── 19. Нет E2E тестов (только ~150 unit тестов утилит)
├── 20. Super-admin emails захардкожены в constants.ts
├── 21. Debt recalculation запускается у КАЖДОГО клиента
├── 22. Нет soft delete для финансовых документов
└── 23. Нет rate limiting
```
